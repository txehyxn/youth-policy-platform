# 기술 스택 선정 이유

## Backend

### Java 25

선정 이유

* 최신 LTS(Long Term Support) 버전 기반 개발
* 최신 Java 기능 학습 및 활용 경험 확보
* Spring Boot 최신 버전과의 호환성
* 장기 유지보수가 가능한 안정적인 개발 환경 구축
* 개인 프로젝트를 통해 최신 Java 생태계 경험 습득

---

### Spring Boot

선정 이유

* 빠른 프로젝트 구축 가능
* 의존성 관리가 편리함
* 계층형 아키텍처 구현에 적합
* REST API 및 웹 애플리케이션 개발에 최적화
* Spring Security, JPA 등과 높은 호환성

---

### Spring Security

선정 이유

* 인증(Authentication) 및 인가(Authorization) 기능 제공
* 로그인 및 권한 관리 구현 가능
* 세션 기반 인증 구현 가능
* 향후 OAuth2 로그인 확장 가능

---

### Spring Data JPA

선정 이유

* 객체 중심 데이터 접근 가능
* 반복적인 SQL 작성 최소화
* 엔티티 기반 데이터 관리 가능
* 생산성 향상 및 유지보수 용이

---

## Database

### MySQL

선정 이유

* 대표적인 관계형 데이터베이스(RDBMS)
* Spring Boot 및 JPA와 높은 호환성
* 풍부한 자료와 커뮤니티 지원
* 안정적인 데이터 관리 가능

---

## 기술 스택 선택 기준

본 프로젝트는 단순 CRUD 구현이 아닌 정책 조건 모델링과 Eligibility Engine 구현이 핵심이다.

따라서 안정적인 백엔드 개발 환경을 제공하는 Spring Boot와 MySQL을 기반으로 설계하였으며, 최신 Java 25를 사용하여 최신 기술 학습과 실무 역량 향상을 동시에 목표로 한다.

또한 GitHub를 활용하여 설계 과정, 개발 과정, 문제 해결 과정을 지속적으로 기록하여 프로젝트의 성장 과정을 문서화한다.
