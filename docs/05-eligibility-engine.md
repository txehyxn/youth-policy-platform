# Eligibility Engine 설계

## 개요

Eligibility Engine은 사용자의 개인정보와 정책 조건을 비교하여 해당 정책을 신청할 수 있는지 자동으로 판별하는 핵심 기능이다.

본 프로젝트는 단순히 정책 정보를 보여주는 것이 아니라, 사용자의 조건을 기준으로 정책 신청 가능 여부를 판단하는 것을 목표로 한다.

---

## 판별 결과

Eligibility Engine은 정책별로 다음 세 가지 결과를 반환한다.

| 결과             | 설명       |
| -------------- | -------- |
| ELIGIBLE       | 신청 가능    |
| NOT_ELIGIBLE   | 신청 불가능   |
| NEED_MORE_INFO | 추가 정보 필요 |

---

## 입력 데이터

사용자 정보는 `user_profiles` 테이블에 저장된다.

예시 입력값:

| 항목                | 예시       |
| ----------------- | -------- |
| age               | 26       |
| region            | 수원       |
| household_size    | 1        |
| monthly_income    | 1800000  |
| income_percent    | 82       |
| employment_status | EMPLOYED |
| student_status    | false    |
| house_owner       | false    |

---

## 정책 조건 데이터

정책 조건은 `benefit_conditions` 테이블에 저장된다.

예시:

| benefit_id | field_name     | operator | value    | required |
| ---------- | -------------- | -------- | -------- | -------- |
| 1          | age            | >=       | 19       | true     |
| 1          | age            | <=       | 34       | true     |
| 1          | region         | IN       | 수원,서울,경기 | true     |
| 1          | house_owner    | =        | false    | true     |
| 1          | income_percent | <=       | 60       | true     |

---

## 판별 흐름

```text
사용자 정보 입력
↓
중위소득 비율 계산
↓
정책 조건 조회
↓
사용자 정보와 정책 조건 비교
↓
신청 가능 / 신청 불가능 / 추가 정보 필요 반환
```

---

## 판별 기준

### 1. 모든 필수 조건을 충족하는 경우

결과:

```text
ELIGIBLE
```

예시:

```text
나이 조건 충족
지역 조건 충족
무주택 조건 충족
소득 조건 충족
```

---

### 2. 하나 이상의 필수 조건을 충족하지 못한 경우

결과:

```text
NOT_ELIGIBLE
```

예시:

```text
중위소득 기준 초과
현재: 82%
기준: 60%
```

---

### 3. 필수 조건 판단에 필요한 사용자 정보가 없는 경우

결과:

```text
NEED_MORE_INFO
```

예시:

```text
학생 여부 정보가 필요합니다.
```

---

## 지원 연산자

| 연산자    | 설명                  |
| ------ | ------------------- |
| =      | 값이 같음               |
| !=     | 값이 다름               |
| >      | 기준값보다 큼             |
| >=     | 기준값 이상              |
| <      | 기준값보다 작음            |
| <=     | 기준값 이하              |
| IN     | 여러 값 중 하나에 포함       |
| NOT_IN | 여러 값 중 하나에도 포함되지 않음 |

---

## 예시: 청년월세지원

### 사용자 정보

| 항목             | 값     |
| -------------- | ----- |
| age            | 26    |
| region         | 수원    |
| house_owner    | false |
| income_percent | 54    |

### 정책 조건

| 조건     | 기준            |
| ------ | ------------- |
| 나이     | 19세 이상 34세 이하 |
| 거주 지역  | 경기            |
| 무주택 여부 | 무주택           |
| 중위소득   | 60% 이하        |

### 판별 결과

```text
ELIGIBLE
```

사유:

```text
나이 조건 충족
지역 조건 충족
무주택 조건 충족
중위소득 기준 충족
```

## 설계 의도

정책마다 자격 조건이 다르기 때문에 조건을 코드에 직접 작성하면 새로운 정책이 추가될 때마다 코드 수정이 필요하다.

이를 해결하기 위해 정책 조건을 데이터베이스에 저장하고, Eligibility Engine이 조건을 동적으로 조회하여 판별하도록 설계하였다.

이 구조를 통해 다음과 같은 장점을 얻을 수 있다.

* 정책 추가 시 코드 수정 최소화
* 조건 변경 시 데이터 수정만으로 대응 가능
* 정책별 판별 사유 제공 가능
* 향후 공공 API 연동 시 확장 가능
* 데이터 기반 정책 분석 기능으로 확장 가능

---

## 핵심 설계 철학

본 프로젝트의 핵심은 모든 정책 데이터를 완벽히 수집하는 것이 아니라, 새로운 정책이 쉽게 추가되고 판별될 수 있는 구조를 만드는 것이다.

따라서 Eligibility Engine은 본 프로젝트의 가장 중요한 기능이며, 단순 CRUD 프로젝트와 차별화되는 핵심 요소이다.
